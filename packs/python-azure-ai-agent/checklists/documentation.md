# Documentation Checklist

## Sync with Code Changes
- [ ] Are README/docs updated when public APIs change?
- [ ] Are new features or commands documented?
- [ ] Are removed/deprecated features marked or removed from docs?
- [ ] Are configuration options and environment variables documented?

## Accuracy
- [ ] Are code examples in docs still accurate after changes?
- [ ] Do installation/setup instructions reflect current requirements?
- [ ] Are version numbers and compatibility info current?
- [ ] Are screenshots or diagrams still accurate (if applicable)?

## Completeness
- [ ] Are all public functions/classes/modules documented?
- [ ] Are error messages and troubleshooting steps documented?
- [ ] Are edge cases and limitations documented?
- [ ] Is there a changelog entry for user-facing changes?

## Discoverability
- [ ] Is new documentation linked from relevant places (README, index)?
- [ ] Are related docs cross-referenced appropriately?
- [ ] Is the documentation organized logically?
- [ ] Can users find answers to common questions?
